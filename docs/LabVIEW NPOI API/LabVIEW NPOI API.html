Overview

Many a time, a software application is required to generate reports in Microsoft Excel file format. Sometimes, an application is even expected to receive Excel files as input data.

What is LabVIEW NPOI?

LabVIEW NPOI is ab API that allows programmers to create, modify, and display MS Office files using LabVIEW programs. It contains classes and methods to decode the user input data or a file into MS Office documents.

It attempts to provide an high-level API for an easy and fast intagration by handling the interface to the .NET version of the POI Java project called NPOI.

Prerequisites

  • LabVIEW 2017 (Full,Pro or Community) or later.
  • Operating Systems: Windows 10 (8 but not tested on it).
  • .NET Standard 2.0 (.NET Core).
  • .NET Framework 4.0 and above.

Installation
  1. Download the latest version of the LabVIEW NPOI VI Package from the release page.
  2. Use VIPM to install it on your LabVIEW version (2017+).

Componant API

The LabVIEW NPOI API exposes properties and methods to an abstract LVClass called Document. This allows us to have a unique manner to manipulate MS Office document, hiding the complexity of the underneath specific components for each file formar (*.doc, *.docx, *.xls,*.xlsx). This Document class is a the top of the class hierarchy, that helps us to ensure reliability , scalability, maintainability of the whole library.

The LabVIEW NPOI dynamically handle the file format, based on the file extension that if pass through, to load the propper type of class. This will help the user to carry out tasks as efficiently as possible and to keep focuss on its most important topics.

Below if the class diagran show the class hierarchy for decoding the Microsoft Office format.

The LabVIEW NPOI also provides an collection of classes for text formatting purpose. It integrates predefined styles such as heading from H1 to H3 and others and will let the user to easily create his owns.
